# ExecPlan: СЭМД Протокол консультации (Редакция 7)

Этот документ является живым планом реализации СЭМД Протокол консультации версии 290 (Редакция 7). Он ведется в соответствии с требованиями `PLANS.md`.

## Цель / Общая картина

Реализация генератора медицинского документа "Протокол консультации" (CDA R2). Результатом является веб-интерфейс, который автоматически формирует форму ввода на основе Pydantic-моделей и генерирует валидный XML-документ. Врач сможет заполнить данные о приеме, диагнозах и назначениях, а система сформирует структурированный документ для передачи в РЭМД.

## Контекст и ориентация

Текущая реализация базируется на следующих компонентах:
- **Registry**: `semd_engine/registry/v290_r7/` — папка со спецификой данного СЭМД.
- **Schema**: `semd_engine/registry/v290_r7/schema.py` — Pydantic модели с UI-метаданными.
- **Templates**: `semd_engine/registry/v290_r7/templates/main.xml.j2` — основной Jinja2 шаблон.
- **Web**: `main.py` — FastAPI сервер для демонстрации и тестирования.
- **Core Engine**: `semd_engine/core/` — общие компоненты (рендерер, макросы, типы данных HL7).

## Surprises & Discoveries

- **Наблюдение**: Браузеры не отправляют значение чекбокса/переключателя, если он выключен. Это приводило к `Missing Field` ошибке в Pydantic.
  - *Доказательство*: При отправке формы без нажатого Switch поле `is_nsi` отсутствовало в `request.form()`.
- **Наблюдение**: Специфические поля HL7 CD (Concept Descriptor) требуют обязательного наличия `code` и `codeSystem`, иначе макросы рендеринга ломаются.
- **Наблюдение**: Функция `flat_to_nested` должна явно игнорировать пустые строки `""` для опциональных числовых полей, чтобы Pydantic мог использовать `None`.

## Decision Log

- **Решение**: Установить `is_nsi: bool = False` в моделях.
  - *Обоснование*: Гарантирует наличие значения даже при "пустом" ответе от веб-формы, упрощая валидацию.
  - *Дата*: 2025-04-21
- **Решение**: Использовать `Annotated` в Pydantic для хранения UI-метаданных.
  - *Обоснование*: Позволяет держать описание интерфейса (лейблы, компоненты) максимально близко к логике данных, обеспечивая Single Source of Truth.
  - *Дата*: 2025-04-21
- **Решение**: Хранить ExecPlans в отдельной директории `execplans/`.
  - *Обоснование*: Улучшает навигацию и отделяет документацию процесса от кода движка.
  - *Дата*: 2025-04-21

## Валидация и приемка

### Автоматические тесты
Запуск всех тестов проекта:
```bash
uv run pytest tests/test_web_interface.py tests/test_pydantic_flow.py
```
Ожидаемый результат: `5 passed`.

### Ручная проверка
1. Запустить сервер: `uv run python main.py`
2. Открыть `http://localhost:8001`
3. Заполнить форму (включая секцию назначений).
4. Нажать "Сгенерировать".
5. Убедиться, что в XML присутствует секция `<substanceAdministration>` с введенными данными.
