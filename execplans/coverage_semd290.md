## Цель / Общая картина
Цель задачи — гарантировать, что все поля, описанные в Pydantic моделях для СЭМД 290 Р7 (Протокол консультации), корректно рендерятся в XML и проходят валидацию по XSD и Schematron. 
Результатом будет набор тестов, который пошагово проверяет каждое поле, выявляя ошибки в генераторе или моделях.

## Контекст и ориентиры
- **СЭМД 290 Р7**: Протокол консультации (Редакция 7).
- **Модели данных**: `/home/dmitry/projects/remd/semd_engine/registry/v290_r7/schema.py` (бизнес-логика) и `/home/dmitry/projects/remd/semd_engine/registry/v290_r7/context.py` (контекст рендеринга).
- **Шаблон**: `/home/dmitry/projects/remd/semd_engine/registry/v290_r7/templates/main.xml.j2`.
- **Валидаторы**: XSD и Schematron находятся в `docs/semd/290/...`.
- **Движок**: `semd_engine/core/renderer.py` использует Jinja2.

## План работы
1.  **Базовая верификация**: Запустить существующие тесты `tests/semd290/test_validation.py`, чтобы убедиться в стабильности текущего состояния.
2.  **Создание теста покрытия**: Создать файл `tests/semd290/test_coverage.py`.
3.  **Итеративная разработка**:
    - Выбираю одну группу полей (например, `patient.insurance_policy`).
    - Добавляю тест, передающий эти данные.
    - Проверяю генерацию и валидацию.
    - Если валидация падает: исправляю шаблон `main.xml.j2` или модель в `schema.py`/`context.py`.
    - Повторяю для всех групп полей.
4.  **Финальная проверка**: Запуск всех тестов проекта.

## Конкретные шаги
- Запуск тестов: `uv run pytest tests/semd290/test_validation.py`
- Создание нового теста: `touch tests/semd290/test_coverage.py`
- Валидация будет происходить внутри тестов через `renderer.validate_xsd` и `renderer.validate_schematron`.

## Валидация и приемка
- Успешное прохождение `test_coverage.py`, который должен включать проверки на присутствие всех ожидаемых XML-тегов и атрибутов.
- Отсутствие ошибок XSD и Schematron при использовании максимально заполненного набора данных.

## Идемпотентность и восстановление
- Тесты можно запускать неограниченное количество раз. 
- Изменения в коде генератора должны быть обратно совместимыми или исправлять явные несоответствия схемам.

## Интерфейсы и зависимости
- `ConsultationProtocolV7RenderContext` — основная входная модель.
- `CDARenderer` — класс для рендеринга и валидации.
